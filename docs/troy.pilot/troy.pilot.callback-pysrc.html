  8      Callback base class. 
  9   
 10      All stateful objects of the pilot API allow to register a callback for 
 11      'state' (and 'state_detail') attribute changes.  That callback is a class 
 12      instance which inherits this callback base class, and implements (overloads) 
 13      the cb() method.  The cb() method is then invoked whenever the TROY 
 14      implementation is notified of a state change of the monitored object. 
 15   
 16      The cb instance receives three parameters upon invokation: 
 17   
 18        - obj:    the watched object instance 
 19        - member: the watched attribute ('state' or 'state_detail') 
 20        - value:  the new value of the watched attribute 
 21   
 22      If cb() returns True, the callback will remain registered after invocation, 
 23      to monitor the object for the next subsequent state change.  On False, the 
 24      callback will not be called again. 
 25   
 26      Only one callback can be registered for any given object instance at any 
 27      given time.  If 'set_callback()' is called twice, the earlier callback is 
 28      unregistered. 
 29   
 30      To register a callback on a object instance, use:: 
 31   
 32        class MyCallback (troy.pilot.Callback) : 
 33              
 34          def __init__ (self, msg) : 
 35            self.msg_ = msg 
 36   
 37          def cb (self, obj, member, value) : 
 38            print " %s\\n %s (%s) : %s"  %  self.msg_, obj, member, value 
 39   
 40          def main () : 
 41             
 42            cpd = troy.pilot.compute_pilot_description () 
 43            cps = troy.pilot.compute_pilot_service () 
 44            cp  = cps.create_pilot (cpd) 
 45   
 46            mcb = MyCallback ("Hello Pilot, how is your state?") 
 47   
 48            cp.set_callback ('state', mcb) 
 49
