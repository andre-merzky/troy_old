 31        - state:

 33          type: L{troy.pilot.State} (enum)

 35        - state_detail:

 37          interpreted by Troy, and is up to the backend pilot framework. 
 38          type: string 
 39

 41          The ComputePilotDescription used to create this pilot.  That description 
 42          is not guaranteed to be available, nor is it guaranteed to be complete 
 43          -- in particular for reconnected pilots.  Its existence and completeness 
 44          depends on the ability to inspect backend pilot instances. 
 45          type: L{troy.pilot.ComputePilotDescription} 
 46

102          Keyword arguments: 
103          cpd -- A ComputePilotDescription 
104   
105          The reinitialize method is intended to change the amount of resources 
106          available to a pilot, without changing the pilot's state otherwise.  The 
107          method can be called in any non-final state, and will automatically move 
108          the pilot to 'Pending' state (if successful).  If reinitialization fails 
109          (for example, because no suitable resources have been found), the pilot 
110          is moved to 'Failed' state. 
111   
112          It is up to the backend is the pilot's ID changes during the call to 
113          reinitialize(), but the implementation SHOULD attempt to keep the ID 
114          constant. 
115

117          may achieve this by aborting the units, and resubmitting them to the 
118          re-initialized pilot.
