 10      """  ComputeUnitService (CUS) 
 11   
 12          The ComputeUnitService is the application's interface to submit 
 13          ComputeUnits to Pilot-Managers in the P* Model.  The PilotManagers are 
 14          represented by L{ComputePilotService} instances -- those instances can 
 15          be added to L{ComputeUnitService} instances, and are then used for 
 16          scheduling L{ComputeUnit}s. 
 17   
 18          The implementation of the ComputeUnitService is generally within 
 19          Troy -- as such, it represents and facilitates the application level 
 20          scheduling capabilities of Troy.  The CUS will use Troy level or 
 21          application provided scheduling algorithms to control the distribution 
 22          of the L{ComputeUnit}s over the set of participating 
 23          L{ComputePilotService}s and their L{ComputePilot}s. 
 24   
 25          Properties:: 
 26   
 27            - id: 
 28              The returned ID can be used to connect to the CUS instance later 
 29              on, for example from within a different application instance. 
 30              type: string (url)
