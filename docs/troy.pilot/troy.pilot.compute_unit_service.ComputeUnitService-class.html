ComputeUnitService (CUS)

The ComputeUnitService is the application's interface to submit ComputeUnits to Pilot-Managers in the P* Model. The PilotManagers are represented by ComputePilotService instances -- those instances can be added to ComputeUnitService instances, and are then used for scheduling ComputeUnits.

The implementation of the ComputeUnitService is generally within Troy -- as such, it represents and facilitates the application level scheduling capabilities of Troy. The CUS will use Troy level or application provided scheduling algorithms to control the distribution of the ComputeUnits over the set of participating ComputePilotServices and their ComputePilots.

Properties:

 - id:
   The returned ID can be used to connect to the CUS instance later
   on, for example from within a different application instance.
   type: string (url)

list_units(self)

source code 

list managed ComputeUnits.

Return value: A list of ComputeUnit IDs

The returned list can include units which have not been created by this CUS instance. The list may be incomplete, and may not include units created by the CUS. There is no guarantee that units in the returned list can in fact be reconnected to. Also, an inclusion in the list does not have any indication about the respective unit's state.
