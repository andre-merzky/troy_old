DataUnitService (DUS)

The DataUnitService is the application's interface to submit DataUnits to the Pilot-Manager in the P* Model. The PilotManagers are represented by DataPilotService instances -- those instances can be added to DataUnitService instances, and are then used for scheduling DataUnits.

The implementation of the DataUnitService is generally within Troy -- as such, it represents and facilitates the application level scheduling capabilities of Troy. The DUS will use Troy level or application provided scheduling algorithms to control the distribution of the DataUnits over the set of participating DataPilotServices and their DataPilots.

Properties:

 - id:
   The returned ID can be used to connect to the DPs instance later
   on, for example from within a different application instance.
   type: string (url)

list_data_units(self)

source code 

list managed DataUnits.

Return value: A list of DataUnit IDs

The returned list can include units which have not been created by this DUS instance. The list may be incomplete, and may not include units created by the DUS. There is no guarantee that units in the returned list can in fact be reconnected to. Also, an inclusion in the list does not have any indication about the respective unit's state.
