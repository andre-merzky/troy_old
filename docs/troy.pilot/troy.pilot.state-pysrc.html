 6 -class State (): 

 7      """ 
 8      The State class is really an enum, or what python uses as an enum. 
 9      That enum describes the state model for various components of the P* model, 
10      and thus of the pilot API:: 
11   
12          troy.pilot.compute_pilot 
13          troy.pilot.compute_unit 
14          troy.pilot.data_pilot 
15          troy.pilot.data_unit 
16          troy.pilot.compute_data_pilot 
17          troy.pilot.compute_data_unit 
18   
19      Those components have the following life cycle:   
20   
21      An api instance is created via a creation request, like:: 
22   
23          cpd = troy.pilot.compute_pilot_description () 
24          cps = troy.pilot.compute_pilot_service () 
25          cp  = cps.create_pilot (cpd) 
26   
27      That compute pilot has, at this point, the state 'New', so the Troy 
28      implementation has accepted the creation request, but it was not yet passed 
29      on to a backend which actually instantiates a compute pilot on some 
30      resource. 
31   
32      Once the request is actually passed on to the backend, and was accepted by 
33      it, the cp will have 'Pending' state.  That state will eventually transition 
34      to 'Running', when the backend actually instantiated the compute pilot 
35      process/agent/...  Resources are only consumed during the 'Running' state,  
36   
37      There are three options to leave the 'Running' state: 'Done', as successful 
38      and planed end of operation of the instance; 'Failed', as premature or 
39      unexpected end of operation (e.g. due to an error condition); and 
40      'Canceled', as premature and expected end of operation, due to user request. 
41   
42   
43      Some service objects of the pilot API may also have state, but in general 
44      only expose 'Running' and 'Unknown' (i.e. not-Running).  Those service 
45      states may disappear in the future.   
46   
47   
48      Note that all stateful objects of the Pilot API will, additionally to the 
49      state enum, expose a 'state_detail' (string), which is, in general, the 
50      native backend state. 
51      """ 
52   
53      Unknown  = "Unknown"    """ The state of the backend instance is not known.  Duh! """ 
54      New      = "New"        """ The creation request was accepted, but did not yet reach the backend. """ 
55      Pending  = "Pending"    """ The backend accepted a creation request, but did not yet enact it. """ 
56      Running  = "Running"    """ The backend instance is created and active. """ 
57      Done     = "Done"       """ The backend instance finished successfully. """ 
58      Canceled = "Canceled"   """ The backend instance finished due to a user request. """ 
59      Failed   = "Failed"     """ The backend instance finished on an error condition. """
