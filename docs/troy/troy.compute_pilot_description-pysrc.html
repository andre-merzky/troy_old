  9      """ ComputePilotDescription. 
 10   
 11      A ComputePilotDescription (CPD) describes a L{ComputePilot} to be 
 12      submitted to a resource.  A well defined set of attributes can be set on 
 13      a CPD to specify the pilot's properties, and the resource requirements 
 14      for the pilot: 
 15   
 16        - 'size': 
 17            - minimum number of processes the pilot is expected to manage at any 
 18              point in time (usually translates to number of CPU process slots 
 19              managed by the pilot). 
 20            - type   : integer 
 21            - default: 1 
 22   
 23        - 'environment': 
 24            - a set of key/value pairs, to be exported into the environment of 
 25              L{ComputeUnit}s on that pilot. 
 26            - type   : dictionary 
 27            - default: {} 
 28   
 29        - 'queue': 
 30            - the name of the queue the pilot should be submitted to. 
 31            - type   : string 
 32            - default: "" 
 33   
 34        - 'project': 
 35            - the project ID used for accounting of the pilot's resource 
 36              consumption. 
 37            - type   : string 
 38            - default: "" 
 39   
 40        - 'candidate_hosts': 
 41            - a list of hostnames on any of which the pilot may be operate. 
 42            - type   : string 
 43            - default: "" 
 44   
 45        - 'cpu_architecture': 
 46            - the CPU architecture the pilot is expected to run on.. 
 47            - type   : string, values as defined by GLUE.v2 
 48            - default: "" 
 49   
 50        - 'operating_system_type': 
 51            - the operating system the pilot is expected to run on. 
 52            - type   : string, values as defined by GLUE.v2 
 53            - default: "" 
 54   
 55        - 'total_physical_memory': 
 56            - the maximum amount of memory the pilot is expected to allocate, in 
 57              total (in MByte). 
 58            - type   : int 
 59            - default: "" 
 60   
 61        - 'total_cpu_time': 
 62            - the maximum number of CPU hours the pilot is expected to consume, in 
 63              total. 
 64            - type   : float (number of CPU hours) 
 65            - default: "" 
 66   
 67        - 'wall_time_limit': 
 68            - the maximum time the pilot is expected to operate, in hours. 
 69              If unspecified, the pilot is assumed to live forever. 
 70            - type   : float (number of hours) 
 71            - default: "" 
 72   
 73        - 'start_time': 
 74            - the point in time when the pilot is expected to become active. 
 75            - type   : time (number of seconds since epoch) or class time.struct_time 
 76            - default: "" 
 77   
 78        - 'contact': 
 79            - a (email, sms, IM) contact URL to notify on pilot state changes. 
 80            - type   : string 
 81            - default: "" 
 82   
 83   
 84   
 85      Affinity: 
 86   
 87      These labels should get assigned by the backend, but are exposed on API 
 88      level for the benefit of application level schedulers:: 
 89   
 90        - 'affinity_datacenter_label': 
 91            - pilots sharing the same label are located in the same data 
 92              center          . 
 93            - type   : string 
 94            - default: "" 
 95   
 96        - 'affinity_machine_label': 
 97            - pilots sharing the same label are located on the same machine. 
 98            - type   : string 
 99            - default: "" 
100   
101   
102      Note that the CPD does not describe how the pilot is instantiated in 
103      detail (e.g.executable name of pilot instance) -- that is left to the 
104      backend to decide, and may (or may not) be configurable out-of-band. 
105   
106      """
