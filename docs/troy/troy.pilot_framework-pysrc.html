 11 -class PilotFramework (Base) : 

 12      """   
 13      PilotFramework (PF) 
 14   
 15      A PF acts as the interface to an underlying pilot job framework -- that 
 16      backend creates and manages pilot instances (L{ComputePilot}, L{DataPilot}), 
 17      and can schedule work unit amongst those pilots (L{ComputeUnit}, 
 18      L{DataUnit}). 
 19   
 20      Class instances are identified by an url, and can be reconnected to.  A PF 
 21      can be added to a L{Troy} instance, which will then be able to utilize the 
 22      PF's pilots for work unit scheduling and execution. 
 23   
 24      Properties:: 
 25   
 26          - id: 
 27            The returned ID can be used to connect to the PF instance later on, 
 28            for example from within a different application instance. 
 29            Type: String 
 30   
 31          - pilots: 
 32            list of pilot id, representing pilots managed by this pilot framework 
 33            instance 
 34            Type: String list 
 35   
 36          - units: 
 37            A list of L{ComputeUnit} IDs, representing compute units managed by 
 38            this pilot. 
 39            Note: This list may or may not be complete.  In particular, the PF may 
 40            choose to not include units which are already assigned to a specific 
 41            pilot, and thus show up in that pilot's 'units' list. 
 42            Type: String list 
 43   
 44      """
