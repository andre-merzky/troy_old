 15      """   
 16      Scheduler (S) 
 17   
 18      The Troy Scheduler class serves two purposes.  First, it is used as API 
 19      class within Troy, and represents an adaptor level scheduler.  Second, it 
 20      acts as a interface to support the implementation of schedulers on 
 21      application, which can be handed to Troy and can be used in the same way as 
 22      adaptor backed scheduler instances. 
 23   
 24      The class implements exactly one method: L{schedule (self, t, ud)}, which 
 25      accepts a L{Troy} instance and a workload description 
 26      (L{ComputeUnitDescription} or L{DataUnitDescription}).  The Troy instance 
 27      represents the set of resources to be scheduled over, the workload 
 28      description represents the workplan to be scheduled on those resources. 
 29   
 30      The L{schedule()} method MUST throw a L{troy.Exception} if scheduling is not 
 31      possible, i.e. if the scheduler either detects contradictory scheduling 
 32      constraints, or if it detects that applying the scheduling constraints on 
 33      the set of Troy's resources results in an empty set.  If the method can 
 34      schedule the work description for execution, i.e. if it hands it off to one 
 35      of Troy's PilotFrameworks or Pilots, the Scheduler MUST return 
 36      a L{ComputeUnit} or L{DataUnit} instance (depending on the description 
 37      type).  Any other return type MAY result in the work unit being scheduled 
 38      twice.  If the scheduler adds constraints, but can not conclusively schedule 
 39      the unit on a pilot framework or pilot, it MUST return None -- not doing so 
 40      may result in the work unit not being scheduled at all. 
 41   
 42      Properties:: 
 43   
 44          - id: 
 45            The ID can be used to connect to the Scheduler instance later on, for 
 46            example from within a different application instance -- although the 
 47            ability to reconnect is not guaranteed.   
 48            Type: String (url) 
 49   
 50      FIXME: we might want to expose scheduler state and statistics as properties, 
 51      for inspection and profiling? 
 52      """
