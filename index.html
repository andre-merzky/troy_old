API Reference

  • The P* model for pilot abstractions describes how pilot application level resource management and scheduling can be implemented by Pilots, which execute specific, application defined work units. The model explicitly targets the well-known PilotJob pattern, but is in fact generic enough to be also applicable to similar patterns in the Data domain (PilotData / PilotStore), and potentially others. TROY is an implementation of the Pilot-API, which exposes the semantics described by the P* Model, and which can thus be used to interact with any Pilot(Job/Data) implementation which implements the P* Model, or which can be described by the P* Model.

    TROY is an open source project, hosted and developed by the SAGA Project and the RADICAL Group at the Cloud and Autonomic Computing Center at Rutgers University.

  • TROY is available via PyPi and installs in easily with easy_install (Installation instructions) 

    $ easy_install troy
